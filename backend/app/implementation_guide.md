## Database Migration (Alembic)
1. **Initialize Alembic (already done).** The repository contains `alembic.ini` and a migration script at `alembic/versions/0001_create_users_table.py`.
2. **Run the migration** to create the `users` table:
   ```powershell
   alembic upgrade head
   ```
   This reads the `DATABASE_URL` from your `.env` and creates the table with columns:
   - `id` (UUID string primary key)
   - `email` (unique)
   - `password_hash`
   - `face_embedding` (encrypted binary)
   - `created_at`, `updated_at`
3. Verify the table exists (e.g., via `psql` or a DB client).

## Running the FastAPI Server
Start the development server with hotтАСreload:
```powershell
uvicorn backend.app.main:app --reload
```
- The API will be reachable at `http://127.0.0.1:8000`.
- Open `http://127.0.0.1:8000/docs` to view the automatically generated OpenAPI Swagger UI.

### Included Routers
| Router | Prefix | Description |
|--------|--------|-------------|
| **auth** | `/auth` | Register & login (email & password). Returns JWT access token. |
| **face** | `/face` | Register face embedding (`/face/register`) and authenticate via face (`/face/login`). Uses InsightFace + AESтАС256 encryption. |
| **user** | `/user` | Profile CRUD (`GET /user/me`, `PATCH /user/me`, `DELETE /user/me`). Requires Bearer token. |
| **admin** | `/admin` | AdminтАСonly: list all users, delete any user. Requires JWT with `role: "admin"`. |
| **analytics** | `/analytics` | Placeholder usage statistics (`/analytics/usage`). |

## Troubleshooting & Tips
- **Permission errors when running commands**: Ensure PowerShell is launched with *Administrator* rights or that the workspace folder is not readтАСonly.
- **`alembic upgrade` fails**: Verify `DATABASE_URL` is correct and the database is reachable. Use `psql` or a client to check connectivity.
- **Face embedding extraction raises `No face detected`**: Make sure the uploaded image contains a clear, frontal face. The default InsightFace model works best with highтАСresolution images.
- **JWT token validation errors**: Confirm the `JWT_SECRET_KEY` in `.env` matches the one used when creating tokens.
- **Encryption key length**: Must be exactly 32 bytes before Base64 encoding (256тАСbit AES key). Regenerate if you see `Invalid key length` errors.
- **Running tests on Windows**: `pytest-asyncio` works outтАСofтАСtheтАСbox, but if you hit `Event loop is closed` errors, add the following at the top of `conftest.py`:
  ```python
  import asyncio
  asyncio.set_event_loop_policy(asyncio.WindowsSelectorEventLoopPolicy())
  ```
